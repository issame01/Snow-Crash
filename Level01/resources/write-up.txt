Vulnerability type:

- Sensitive data exposure
- Improper password storage
- Broken authentication configuration

Note: This is an Old-school UNIX mistake. Normally, modern Linux systems look like this==> username:x:UID:GID:...

The x means: The real password hash is stored in /etc/shadow (root-only)
But in our case, flag01 has a hash directly in /etc/passwd, which is world-readable. ===> flag01:42hDRfypTqqnw:3001:3001::/home/flag/flag01:/bin/bash
This is a classic misconfiguration vulnerability.


What is 42hDRfypTqqnw?

This string is a DES-based Unix password hash, identifiable by its length (13 chars)

- crackable offline
- No bruteforcing the system.

    echo 'flag01:42hDRfypTqqnw' > flag01.hash

Summary:

The system was misconfigured: the password hash of flag01 was stored in /etc/passwd instead of /etc/shadow. 
Since /etc/passwd is world-readable, I extracted the DES hash and cracked it offline using John the Ripper.
